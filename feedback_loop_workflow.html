🔁 Feedback Loop Workflow

Atlas — Unsatisfactory Output Alarm | v1 brainstorm draft

Lemar triggers a task or prompt
🤖 Atlas / Samira generates output (email draft, analysis, Monday task, etc.)
📬 Output delivered to Lemar (Slack reply · Monday item · Gmail draft)
⚖️ Lemar evaluates output Is the output satisfactory?
✓ Yes
✅ Done Log to #reports
✗ No — alarm
🚨 Lemar sends alarm signal Signal method: TBD — see parameters
🔁 Correction loop triggered Retry with context · escalate · clarify
⚠️ Retry count > max retries? If no → back to output delivery ↑
🛑 Post to #action-items STUCK — needs Lemar's decision
↑ loops back to output delivery on retry
Parameters to Define — Brainstorm
1. Signal Method
How does Lemar send the alarm?
• Emoji reaction (:x: / 👎 / custom)
• Keyword in reply ("redo", "wrong", "off")
• Rating in reply (1–5, below threshold = alarm)
• Custom slash command / reaction combo
2. Signal Content
What info should the alarm carry?
• Freeform reply text (just say what's wrong)
• Structured fields: what's wrong / severity / suggested fix
• Signal alone (system infers from context)
• Required: what was wrong + what you expected
3. Alarm Threshold
What qualifies as unsatisfactory?
• Any negative signal from Lemar
• Only explicit alarm triggers (not minor tweaks)
• Severity score above N (1–5 scale)
• Specific output types only (emails, financials)
4. Correction Strategy
What happens when an alarm fires?
• Auto-retry same prompt (no changes)
• Retry with alarm context appended to prompt
• Ask Lemar a clarifying question first
• Escalate to a different model or skill
5. Max Retries
How many loops before escalating?
• 1 retry, then escalate
• 2 retries (default)
• 3 retries (admin-3x pattern, already in Samira)
• No limit with human-in-loop gate each time
6. Alarm Landing Zone
Where does the alarm notification post?
• Reply in same thread (inline)
• New post in #action-items
• DM to Lemar
• Monday item update on the source task
• All of the above
7. Failure Archive
Where do unsatisfactory outputs get logged?
• #reports with "ALARM" or "FAILED" tag
• Monday item update on the source task
• Dedicated #feedback channel
• None (ephemeral, just retry)
8. Scope
Which outputs can trigger an alarm?
• All Samira outputs
• Email drafts only
• Monday items only
• Scoped by channel (#emails, #admin, etc.)
• Any output Lemar explicitly marks
